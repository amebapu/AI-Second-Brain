# Skill: knowledge-organizer

## 说明
将 Inbox 中的文章和笔记整理到 Knowledge 知识库，建立关联，形成结构化知识体系。

## Vault 路径
`~/shared-vault/`

## 指令

### 整理知识
当用户说"整理知识"、"把收藏的文章融入知识库"、"帮我整理一下"等：

1. **扫描待整理内容**：
   - `00_Inbox/articles/` 中 status 不为 processed 的文章
   - `00_Inbox/thoughts/` 中的灵感笔记
   
2. **分析与归类**：
   - 识别文章的核心主题
   - 匹配 `02_Knowledge/` 中已有的分类目录
   - 如果没有匹配的分类，建议创建新分类

3. **知识融合**（核心步骤）：
   
   **情况A — 已有相关知识笔记**：
   - 读取已有的知识笔记
   - 将新内容融入已有笔记（补充、更新、修正）
   - 更新 YAML 中的 `last_updated` 和 `sources`
   - 添加 `[[双链]]` 关联
   
   **情况B — 全新主题**：
   - 使用 `_templates/knowledge-note.md` 模板创建新笔记
   - 文件路径：`02_Knowledge/分类/主题名.md`
   - 提取核心概念、关键知识点
   - 建立与已有笔记的 `[[双链]]` 关联

4. **更新原始文章状态**：
   - 将已整理的文章 YAML status 改为 `processed`

5. **vault-sync**

### 知识查询
当用户问"关于xxx的知识"、"我之前学过xxx吗"等：

1. 在 `02_Knowledge/` 中搜索相关文件
2. 也搜索 `00_Inbox/articles/` 中可能相关的未整理文章
3. 综合展示已有知识 + 提醒还有未整理的相关文章

### 知识图谱概览
当用户说"看看知识库"、"知识库有什么"等：

1. 列出 `02_Knowledge/` 的目录结构
2. 统计各分类下的笔记数量
3. 用 Dataview 查询语法建议用户在 Obsidian 中查看更详细的统计

### 关联建议
定期（或用户主动触发）：

1. 扫描 `02_Knowledge/` 中没有 `[[双链]]` 的孤立笔记
2. 分析内容相关性
3. 建议添加关联

## 知识分类预设
初始分类（可随时扩展）：
```
02_Knowledge/
├── AI/            # AI、LLM、RAG、Prompt 等
├── 编程/          # 编程语言、框架、算法
├── 架构/          # 系统设计、微服务、云原生
├── 前端/          # 前端框架、CSS、性能优化
├── DevOps/        # CI/CD、容器、部署
├── 产品/          # 产品思维、用户体验
├── 方法论/        # 学习方法、思维模型、效率
└── 其他/          # 未分类
```

## 双链规范
- 引用格式：`[[分类/笔记名]]`，如 `[[AI/RAG检索策略]]`
- 新建笔记时主动搜索可能相关的已有笔记
- 关联至少 1-2 个已有笔记（如果有的话）

## 注意事项
⚠️ **此 Skill 计算量较大**，个人 Agent 应建议用户转发给公司 Agent 执行。
