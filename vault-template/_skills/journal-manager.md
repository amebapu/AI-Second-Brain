# Skill: journal-manager

## 说明
管理 Obsidian vault 中的日志（01_Journal/），支持增删查改，删除走软删除。

## Vault 路径
`~/shared-vault/`

## 指令

### 写日志 / 追加日志
当用户说"记日志"、"写日志"、"记录"等：

1. 确定日期（默认今天），文件名：`01_Journal/YYYY-MM-DD.md`
2. 如果文件不存在，用模板创建：
   ```yaml
   ---
   date: "YYYY-MM-DD"
   type: journal
   tags: [daily]
   created_by: "{{当前agent名}}"
   ---
   # YYYY-MM-DD 日志
   
   ## 记录
   
   - HH:MM | 用户说的内容
   ```
3. 如果文件已存在，在 `## 记录` 下追加新条目：
   ```
   - HH:MM | 新内容
   ```
4. 操作后执行 vault-sync 的 sync-push

### 查看日志
当用户说"看日志"、"查日志"、"今天记了什么"等：

1. 读取对应日期的 `01_Journal/YYYY-MM-DD.md`
2. 简洁展示内容
3. 如果用户说"最近的日志"，列出最近 7 天的文件名

### 修改日志
当用户说"改一下日志"、"修改日志里的xxx"等：

1. 先展示当前日志完整内容
2. 确认要修改的部分
3. 执行修改
4. vault-sync

### 删除日志内容
⚠️ **软删除流程**（严格执行）：

当用户说"删掉"、"删除"日志中的内容：

1. **展示**：先展示要删除的内容
2. **确认**：明确问用户"确认删除以下内容吗？"
3. **等待**：必须等到用户回复"确认"、"是"、"对"
4. **备份**：将被删除的内容追加到 `05_Archive/deleted-journal-entries.md`
5. **执行删除**
6. **vault-sync**

### 删除整个日志文件
如果用户要删除整天的日志：
1. 展示文件内容
2. 确认
3. 将文件移动到 `05_Archive/journal/YYYY-MM-DD.md`
4. vault-sync
5. 告知用户：文件已移到 Archive，30天内可恢复

## 注意事项
- 时间戳使用 24 小时制
- 日志按时间顺序追加，不倒序
- 标签可以根据内容自动打上，如 `#工作` `#学习` `#生活`
